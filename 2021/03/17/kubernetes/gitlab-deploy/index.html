gitlab部署文档

应用版本

  • gitlab:sameersbn/gitlab:13.9.3
  • postgresql:sameersbn/postgresql:12-20200524
  • redis:redis:5.0.9

使用docker-compose部署

提前在机器上安装好docker和docker-compose
编写 docker-compose.yaml文件

运行docker-compose up启动gitlab服务
初始化时间大概在3分钟左右，docker-compose ps 看到gitlab应用为health即可
访问localhost:10080访问gitlab网站并且使用前面配置的mutou@admin.com 和 adminpassword进行登录

注意: 期间postgresql会有错误抛出，怀疑是sql重复初始化导致的。对gitlab的运行没有影响

登录账号和密码为 GITLAB_ROOT_EMAIL/GITLAB_ROOT_PASSWORD

CI/CD

安装gitlab-runner

  • 首先拉取并部署gitlab版本对应的runner
  • 进入Runner容器内部

docker exec -it gitlab-runner bash

  • 向gitlab注册runner

完成注册后 exit退出

  • 修改Runner配置文件

vi config/config.toml

修改好后重启Runner
docker restart gitlab-runner

配置openLDAP登录

OpenLDAP是一个开源的LDAP实现,一般用作多个开源软件中的统一登录支持
gitlab也是支持LDAP协议进行登录的,下面我们通过一些配置完成gitlab和openldap的整合

  • 根据已经搭建配置好的LDAP服务器获取以下信息
项目 内容 备注
LDAP 服务IP端口 10.233.11.157:389
LDAP 管理员DN cn=admin,dc=kwl,dc=com
LDAP 管理员密码 password
LDAP 账户搜索域 ou=user,dc=kwl,dc=com
LDAP 在域配置下的账户 uid zsan
LDAP 在域配置下的账户 email zhansan@test.com email是额外字段,如果想要使用邮箱作为账户登录则必填
LDAP 在域配置下的账户 password password
  • 示例配置

我们使用的是k8s部署的gitlab,所以修改应用环境变量示例如下:
这个配置使用的LDAP用户的邮箱地址作为账号登录

环境变量key 环境变量value 备注
LDAP_ENABLED true 开启LDAP认证登录
LDAP_HOST 10.233.11.157 LDAP服务的IP地址
LDAP_LABEL LDAP LDAP登录页的名称
LDAP_METHOD plain 加密方式
LDAP_UID mail 如果想直接使用账户名登录则填uid
LDAP_VERIFY_SSL false 启用SSL认证
LDAP_BIND_DN cn=admin,dc=kwl,dc=com LDAP admin DN
LDAP_PASS password LDAP admin 密码
LDAP_TIMEOUT 10 登录超时时间秒
LDAP_ACTIVE_DIRECTORY false 是否是 Active Directory LDAP 服务
LDAP_BASE ou=user,dc=kwl,dc=com 登录用户的扫描域
LDAP_ALLOW_USERNAME_OR_EMAIL_LOGIN false 是否开启邮箱或用户名登录

gitlab-docker 详细配置列表

  • 验证效果

在修改好环境变量后,等待gitlab Pod自动重启就绪
在gitlab登录页面上切换到LDAP登录卡片
使用LDAP用户邮箱和密码(zhansan@test.com/password)进行登录
可以成功登录即配置成功

  • 问题排查

当出现无法登录或者提示没有权限时善用gitlab应用和ldap服务的日志，特别是ldap会输出UNBIND相关日志，这个对于排查登录问题十分有用！
