Helm V3

Helm V2 到 V3 的变化

  • 删除tiller服务
  • release不再是全局资源，而是存储在各自的命名空间中
  • 把chart中的requirements.yaml内容合并到Chart.yaml中
  • helm install 时需要提供名称，如果需要自动生成则需要使用 - g 参数
  • helm delete 时，默认带上–purge(不保留历史)，如果需要保留历史则需要带上–keep-history命令
  • Values支持Json Schema校验器

Helm V3的部署

在官方git上下载对应服务器版本的Helm 二进制包，解压

tar -zxvf helm-v3.0.0-linux-amd64.tgz

然后把helm 二进制文件移动到/usr/local/bin目录下

mv linux-amd64/helm /usr/local/bin/helm

Helm V3的基本使用

与v2版本不同的是在完成v3的安装后必须要加入一个镜像仓库，我们这里使用国内的镜像仓库

helm repo add stable https://mirror.azure.cn/kubernetes/charts

添加仓库名称为stable，搜索对应仓库内的chart镜像

helm repo list

更新仓库内的镜像为最新的latest版本

helm repo update

搜索镜像仓库中的chart

helm search repo stable

下面的例子为安装一个mysql的chart镜像

查询stable仓库中mysql镜像的描述

helm show chart stable/mysql

详细描述的查询命令为

helm show all stable/mysql

安装chart

helm install test-mysql stable/mysql -n default

查询镜像的状态

helm status test-mysql

删除镜像

helm uninstall test-mysql
